# Skvare Custom Bootstrap

## Prerequisites

- Node.js version > 20 must be installed on your machine.

## Getting Started

To get started with the project, follow the instructions below.

### Installation

1. npm install

### Scripts
The following scripts are available for running various tasks:

`npm run prebuild`: Removes the contents of the "dist" directory before the build.

`npm run build`: Builds the project in production mode using webpack.

`npm run build:dev`: Builds the project in development mode using webpack.

`npm run start`: Starts the project and watches for changes using webpack.

`npm run start:dev`: Starts the project in development mode and watches for changes using webpack.

### Script Descriptions
`prebuild`: This script is executed before the build script. It clears the contents of the "dist" directory to ensure a clean build.

`build`: This script builds the project in production mode using webpack. It uses the configuration file webpack.config.js and sets the mode to production.

`build:dev`: This script builds the project in development mode using webpack. It uses the configuration file webpack.config.js and sets the mode to development.

`start`: This script starts the project and watches for changes using webpack. It automatically rebuilds the project whenever changes are detected.

`start:dev`: This script starts the project in development mode and watches for changes using webpack. It rebuilds the project in development mode whenever changes are detected.

### Folder Structure
`dist/`: Contains the compiled output files generated by webpack during the build process.

`scss/`: Contains the SCSS stylesheets. SCSS files that start with an underscore _ will be compiled into the global styles.css file, which will be loaded on the site globally. Files that don't start with an underscore _ will be compiled into separate files that should be included in the imba_boostrap.libraries.yml.

`scripts/`: Contains the JavaScript scripts. All directories and scripts, except the plugins/ directory, will be compiled into the global components.js file. The plugins/ directory will not be processed by webpack.

### Project Setup and Build Instructions
To get started, follow the steps below:

1. Clone the repository to your local machine.
2. Install the required dependencies by running `npm install`.
3. Customize the project as needed by modifying the source files located in the appropriate directories.
4. To build the project for development, run the command npm run build:dev. This will generate the development build in the dist directory.
5. To build the project for production, run the command npm run build. This will generate the optimized production build in the dist directory.
6. Once the build process completes, commit the dist directory to the repository using `git add dist` and `git commit -m "[description]"`.
7. Finally, push the changes to your remote repository using git push.
